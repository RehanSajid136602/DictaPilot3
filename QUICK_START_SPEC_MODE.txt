================================================================================
  DICTAPILOT SPEC MODE - QUICK START
================================================================================

NEW FEATURE: Voice-to-Spec Workflow for Agentic Coding

Transform your voice into structured specifications that AI agents can implement.

--------------------------------------------------------------------------------
SETUP (30 seconds)
--------------------------------------------------------------------------------

1. Add to your .env file:

   SPEC_MODE_ENABLED=1
   SPEC_AUTO_DETECT_INTENT=1

2. Restart DictaPilot (if running)

3. Done! You're ready to use spec mode.

--------------------------------------------------------------------------------
TRY IT NOW
--------------------------------------------------------------------------------

Press F9 and say:

  "Start new spec: Add dark mode toggle to settings"

Continue speaking:

  "Context: Users want dark mode for night coding.
   Acceptance criteria: Toggle in settings, persists on reload.
   Constraint: Must support system theme detection."

Send to your IDE:

  "Send to Cursor"

That's it! Your spec is now in Cursor ready for implementation.

--------------------------------------------------------------------------------
VOICE COMMANDS
--------------------------------------------------------------------------------

CREATE SPECS:
  "Start new spec: [title]"
  "Create feature spec: [title]"
  "Create bugfix spec: [title]"

ADD DETAILS:
  "Goal: [description]"
  "Context: [background]"
  "Acceptance criteria: [criterion]"
  "Constraint: [limitation]"
  "File: [path]"

MANAGE:
  "Save spec"
  "Export as [format]"        (standard/openspec/luna/github)
  "Send to [agent]"            (cursor/windsurf/cline/luna)

--------------------------------------------------------------------------------
SUPPORTED IDEs
--------------------------------------------------------------------------------

✓ Cursor      - Saved to ~/.cursor/dictapilot_input.md
✓ Windsurf    - Saved to ~/.windsurf/dictapilot_input.md
✓ Cline       - Saved to ~/.vscode/cline/dictapilot_input.md
✓ Luna Drive  - Sent via API (requires API key)
✓ Custom      - Configure webhook endpoints

--------------------------------------------------------------------------------
EXAMPLE WORKFLOW
--------------------------------------------------------------------------------

You: "Start new spec: User authentication with JWT"

You: "Goal: Implement secure authentication.
      Context: No auth currently, need to protect API.
      Acceptance criteria: Register, login, logout, JWT expires in 24h.
      Constraint: Use bcrypt for passwords.
      Files: app.py, auth.py"

You: "Save spec. Send to Cursor."

✓ Spec saved and sent to Cursor!

--------------------------------------------------------------------------------
EXPORT FORMATS
--------------------------------------------------------------------------------

STANDARD    - General markdown format
OPENSPEC    - OpenSpec workflow format (compatible with OpenSpec tools)
LUNA        - Luna Drive format (compatible with Luna AI)
GITHUB      - GitHub issue/PR format

--------------------------------------------------------------------------------
STORAGE
--------------------------------------------------------------------------------

Specs are automatically saved to:

Linux/macOS:  ~/.local/share/dictapilot/specs/
Windows:      %APPDATA%\DictaPilot\specs\

Each spec includes:
  - spec.md (markdown version)
  - spec.json (structured data)
  - versions/ (version history)

--------------------------------------------------------------------------------
TROUBLESHOOTING
--------------------------------------------------------------------------------

Spec mode not working?
  → Check SPEC_MODE_ENABLED=1 in .env
  → Use explicit trigger: "Start new spec"

Agent not receiving specs?
  → Verify agent is running
  → Check connection in dashboard

Intent misclassified?
  → Use explicit commands
  → Check SPEC_AUTO_DETECT_INTENT setting
